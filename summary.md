# 个人实训总结

## 遇到的最大困难及解决过程

本次实训中遇到的最大困难不是单个页面或接口的编写，而是如何把一个全栈项目从“能写代码”推进到“可以被别人复现、可以部署、可以检查”的状态。最开始只是根据考核要求确定了 Next.js + Flask 的技术路线，但真正落地时需要同时考虑前端路由、后端 API、数据库、环境变量、部署平台、Git 留痕和文档材料。如果只完成页面和接口，项目看起来可以运行，但提交时仍然可能因为缺少部署说明、缺少接口文档、缺少 Prompt 日志或缺少测试记录而失分。

在解决过程中，我先把项目拆成几个明确阶段：项目初始化、后端 API、前端交互、测试验证、部署准备和文档收尾。后端使用 Flask、SQLAlchemy 和 Flask-CORS，实现了任务管理和 Prompt 日志两个核心模块，并提供了健康检查接口。为了让项目既能本地开发又能部署到 Render，数据库连接通过 `DATABASE_URL` 环境变量读取；本地默认使用 SQLite，生产环境使用 Supabase PostgreSQL。前端使用 Next.js App Router，设计了 Dashboard、Tasks、Prompts、Review 四个独立路由，满足前端至少 3 个独立路由的要求。

另一个实际问题是本机环境中的 Next.js 构建验证。当前 Windows/Codex 环境里，`next build` 遇到了 SWC 原生包加载失败的问题。为避免 Vercel 构建阶段依赖后端 API，我将 Dashboard 改成客户端组件，让页面在浏览器运行时请求 API，而不是在构建阶段请求后端。同时使用 TypeScript 的 `tsc --noEmit` 做静态检查，后端则通过 pytest 完成 API 测试。这样虽然本机没有完成 Next build，但可以证明代码类型检查和后端接口逻辑是通过的，最终前端构建以 Vercel 云端结果为准。

## 对 AI 辅助编程的心得体会

这次项目让我认识到，AI 辅助编程最有价值的地方不是简单生成代码，而是帮助我把任务拆解清楚，并持续检查项目是否符合考核标准。比如考核要求里不仅有“前端 3 个路由、后端 3 个 API”，还有线上 Demo、Git 提交记录、README、Prompt 日志、接口文档、Code Review 和个人总结。如果没有先做需求拆解，很容易只关注功能，最后发现材料不完整。

在开发过程中，我让 AI 先阅读考核要求，再设计项目方案和阶段计划，然后逐步生成基础框架、补充 API、完善前端页面、增加测试和整理部署文档。每一步都不是一次性复制完整项目，而是根据当前进度继续推进，并把关键 Prompt 和 AI 回复记录到 `prompt_log.md` 中。这样做的好处是开发过程可以追溯，也能说明每一次 AI 输出对应解决了哪个功能或文件。

同时，我也发现 AI 输出不能不检查就直接使用。例如前端页面中曾经出现 Windows 控制台编码显示异常的问题，如果不及时检查源码和 TypeScript 结果，可能会把不可编译的代码提交进去。后端测试也是必要的，因为接口看起来写完了，不代表创建、更新、删除和查询流程都能正确工作。通过 pytest 覆盖核心 API 后，项目质量更稳定，也更符合工程化要求。

## 个人具体贡献

本项目由个人独立完成。我完成了项目选题、需求拆解、技术栈选择、目录结构设计、Flask 后端 API 开发、Next.js 前端页面开发、数据库连接方案设计、部署配置整理、接口文档编写、Prompt 日志整理、AI Code Review 报告整理和个人总结撰写。

具体来说，后端实现了 `GET /api/health`、`GET /api/tasks`、`POST /api/tasks`、`PATCH /api/tasks/<id>`、`DELETE /api/tasks/<id>`、`GET /api/prompts`、`POST /api/prompts` 等接口。前端实现了 `/`、`/tasks`、`/prompts`、`/review` 四个页面，并在 Tasks 和 Prompts 页面中加入了可操作的表单和列表。工程化方面，项目包含 `.gitignore`、`render.yaml`、`runtime.txt`、`package-lock.json`、`requirements-dev.txt` 和 pytest 测试，方便后续部署和复现。

通过这次实训，我对 AI 辅助编程的理解从“让 AI 写代码”转变为“用 AI 协助完成工程流程”。一个完整项目不仅要能运行，还要能说明为什么这样设计、如何安装、如何部署、如何验证，以及开发过程中如何使用 AI。后续如果继续完善，我会优先补充更完整的页面样式、接口异常处理、部署截图和演示录屏，让项目从可交付进一步提升到更专业的展示状态。
